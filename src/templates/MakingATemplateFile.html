Making a Template Listing File

Template listing files are used as an index for the creation of multiple source files in a package. template listings contain a multiplicity value, source template file, and a destination file pattern.

Example

PER_STRUCT cpp/myfile.cpp -<series_dir>-/-<datatype_name>-.cpp

In the above example, the multiplicity is PER_STRUCT, which means that the template will be used to create a source file for every struct in the MDM. The source file is "myfile.cpp" in the cpp directory (directory trees are assumed to start in the templates directory.) The output files will be named by their structs with a .cpp extension.

Valid multiplicty tags

The following multiplicity tags are recognized:

  • ONCE Creates a single instance of the new file based on the template.
  • PER_MDM Creates a instance of the file for every MDM encountered.
  • PER_STRUCT Creates a file for every struct encountered in each MDM.
  • PER_ENUM Creates a file for every enum encountered in each MDM.

Text Replacement

Every time a -<some_text>- tag is encountered in a template listing file the text and the open/close tags are replaced by text produced in the Methods class that has been selected by the LMCP generation session.
